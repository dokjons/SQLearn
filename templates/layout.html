SELECT

Short Description: Retrieves data from a database.

Long Description: The SELECT statement is used to select data from a database. The data returned is stored in a result table, sometimes called the result set.

SELECT column1, column2 FROM table_name;

INSERT

Short Description: Inserts new records into a table.

Long Description: The INSERT INTO statement is used to insert new records in a table. You can insert data into specific columns or into all columns.

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

UPDATE

Short Description: Modifies existing records.

Long Description: The UPDATE statement is used to modify the existing records in a table. The SET clause specifies which columns to modify and the new values.

UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;

DELETE

Short Description: Deletes existing records.

Long Description: The DELETE statement is used to delete existing records in a table. The WHERE clause specifies which records should be deleted.

DELETE FROM table_name WHERE condition;

CREATE TABLE

Short Description: Creates a new table.

Long Description: The CREATE TABLE statement is used to create a new table in the database. You define the columns and their data types.

CREATE TABLE table_name (PersonID int, City varchar(255), column2 datatype);

DROP TABLE

Short Description: Deletes a table.

Long Description: The DROP TABLE statement is used to delete a table and all its data from the database.

DROP TABLE table_name;

ALTER TABLE

Short Description: Modifies a table.

Long Description: The ALTER TABLE statement is used to add, delete, or modify columns in an existing table.

ALTER TABLE table_name ADD column_name datatype;

DROP DATABASE

Short Description: Deletes a database.

Long Description: The DROP DATABASE statement is used to delete an existing database and all its data.

DROP DATABASE database_name;

CREATE INDEX

Short Description: Creates an index.

Long Description: The CREATE INDEX statement is used to create an index on a table. Indexes are used to retrieve data more quickly.

CREATE INDEX index_name ON table_name (column_name);

DROP INDEX

Short Description: Deletes an index.

Long Description: The DROP INDEX statement is used to delete an index from a table.

DROP INDEX index_name;

CREATE VIEW

Short Description: Creates a view.

Long Description: The CREATE VIEW statement is used to create a view. A view is a virtual table based on the result set of an SQL statement.

CREATE VIEW view_name AS SELECT column1, column2, FROM table_name WHERE condition;

DROP VIEW

Short Description: Deletes a view.

Long Description: The DROP VIEW statement is used to delete a view.

DROP VIEW view_name;

TRUNCATE TABLE

Short Description: Removes all data from a table.

Long Description: The TRUNCATE TABLE statement is used to delete all data inside a table, but not the table itself.

TRUNCATE TABLE table_name;

GRANT

Short Description: Grant privileges to a user.

Long Description: The GRANT statement is used to grant privileges to database users.

GRANT SELECT, INSERT ON table_name TO 'username';

REVOKE

Short Description: Removes privileges from a user.

Long Description: The REVOKE statement is used to remove previously granted privileges from database users.

REVOKE SELECT, INSERT ON table_name FROM 'username';

COMMIT

Short Description: Saves a transaction.

Long Description: The COMMIT statement is used to save all changes made in the current transaction. Once committed, the changes are permanent.

COMMIT;

ROLLBACK

Short Description: Undoes a transaction.

Long Description: The ROLLBACK statement is used to undo changes made in the current transaction. The changes are not saved permanently.

ROLLBACK;

SAVEPOINT

Short Description: Sets a savepoint within a transaction.

Long Description: The SAVEPOINT statement is used to set a savepoint within a transaction. You can roll back to this savepoint if needed.

SAVEPOINT savepoint_name;

RELEASE SAVEPOINT

Short Description: Releases a savepoint.

Long Description: The RELEASE SAVEPOINT statement is used to release a previously set savepoint.

RELEASE SAVEPOINT savepoint_name;

SET TRANSACTION

Short Description: Sets transaction properties.

Long Description: The SET TRANSACTION statement is used to specify characteristics for the transaction.

SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;

UNION

Short Description: Combines results of two or more SELECT statements.

Long Description: The UNION operator is used to combine the result sets of two or more SELECT statements. It removes duplicate records between the combined result sets.

SELECT column1, column2 FROM table1 UNION SELECT column1, column2 FROM table2;

UNION ALL

Short Description: Combines results of two or more SELECT statements, including duplicates.

Long Description: The UNION ALL operator is used to combine the result sets of two or more SELECT statements, including duplicate records.

SELECT column1, column2 FROM table1 UNION ALL SELECT column1, column2 FROM table2;

INTERSECT

Short Description: Returns common records from two SELECT statements.

Long Description: The INTERSECT operator is used to return the common records from two SELECT statements.

SELECT column1, column2 FROM table1 INTERSECT SELECT column1, column2 FROM table2;

EXCEPT

Short Description: Returns records from the first SELECT statement that are not in the second SELECT statement.

Long Description: The EXCEPT (or MINUS in some databases) operator is used to return the records from the first SELECT statement that are not in the second SELECT statement.

SELECT column1, column2 FROM table1 EXCEPT SELECT column1, column2 FROM table2;

INNER JOIN

Short Description: Returns records with matching values in both tables.

Long Description: The INNER JOIN keyword selects records that have matching values in both tables.

SELECT table1.column1, table2.column2 FROM table1 INNER JOIN table2 ON table1.common_column = table2.common_column;

LEFT JOIN

Short Description: Returns all records from the left table and matched records from the right table.

Long Description: The LEFT JOIN (or LEFT OUTER JOIN) keyword returns all records from the left table, and the matched records from the right table. The result is NULL from the right side, if there is no match.

SELECT table1.column1, table2.column2 FROM table1 LEFT JOIN table2 ON table1.common_column = table2.common_column;

DELETE

Short Description: Returns all records from the right table and matched records from the left table.

Long Description: The RIGHT JOIN (or RIGHT OUTER JOIN) keyword returns all records from the right table, and the matched records from the left table. The result is NULL from the left side, if there is no match.

SELECT table1.column1, table2.column2 FROM table1 RIGHT JOIN table2 ON table1.common_column = table2.common_column;

FULL JOIN

Short Description: Returns all records when there is a match in either left or right table.

Long Description: The FULL JOIN (or FULL OUTER JOIN) keyword returns all records when there is a match in either left (table1) or right (table2) table records.

SELECT table1.column1, table2.column2 FROM table1 FULL JOIN table2 ON table1.common_column = table2.common_column;

CROSS JOIN

Short Description: Returns the Cartesian product of two tables.

Long Description: The CROSS JOIN keyword returns the Cartesian product of the two tables. This means that it will return all possible combinations of rows from both tables.

SELECT table1.column1, table2.column2 FROM table1 CROSS JOIN table2;

NATURAL JOIN

Short Description: Returns rows with matching values in both tables and removes duplicate columns.

Long Description: The NATURAL JOIN keyword selects rows with matching values in both tables and removes duplicate columns.

SELECT * FROM table1 NATURAL JOIN table2;

EXISTS

Short Description: Tests for the existence of any record in a subquery.

Long Description: The EXISTS operator is used to test for the existence of any record in a subquery. It returns TRUE if the subquery returns one or more records.

SELECT column1 FROM table1 WHERE EXISTS (SELECT column1 FROM table2 WHERE table1.common_column = table2.common_column);

ANY

Short Description: Returns TRUE if any subquery values meet the condition.

Long Description: The ANY (or SOME) operator returns TRUE if any of the subquery values meet the condition.

SELECT column1 FROM table1 WHERE column1 = ANY (SELECT column1 FROM table2 WHERE condition);

ALL

Short Description: Returns TRUE if all subquery values meet the condition.

Long Description: The ALL operator returns TRUE if all of the subquery values meet the condition.

SELECT column1 FROM table1 WHERE column1 > ALL (SELECT column1 FROM table2 WHERE condition);

IN

Short Description: Allows you to specify multiple values in a WHERE clause.

Long Description: The IN operator allows you to specify multiple values in a WHERE clause. It is a shorthand for multiple OR conditions.

SELECT column1 FROM table_name WHERE column1 IN (value1, value2, ...);

BETWEEN

Short Description: Selects values within a given range.

Long Description: The BETWEEN operator selects values within a given range. The values can be numbers, text, or dates.

SELECT column1 FROM table_name WHERE column1 BETWEEN value1 AND value2;

LIKE

Short Description: Searches for a specified pattern in a column.

Long Description: The LIKE operator is used in a WHERE clause to search for a specified pattern in a column.

SELECT column1 FROM table_name WHERE column1 LIKE pattern;

IS NULL

Short Description: Tests for empty values.

Long Description: The IS NULL operator is used to test for empty values (NULL).

SELECT column1 FROM table_name WHERE column1 IS NULL;

IS NOT NULL

Short Description: Tests for non-empty values.

Long Description: The IS NOT NULL operator is used to test for non-empty values (NOT NULL).

SELECT column1 FROM table_name WHERE column1 IS NOT NULL;

CASE

Short Description: Creates different outputs based on conditions.

Long Description: The CASE statement is used to create different outputs based on conditions. It goes through conditions and returns a value when the first condition is met.

SELECT column1, CASE WHEN condition1 THEN result1 WHEN condition2 THEN result2 ELSE result3 END FROM table_name;
